### 🛡️ Linux HIDS Automation — Real-Time Host Intrusion Detection

<img align="right" src="https://media.giphy.com/media/LaVp0AyqR5bGsC5Cbm/giphy.gif" width="200"/>

> A **Python-based Host Intrusion Detection System** built for Linux environments.
> Monitors system logs in real time, detects brute-force attacks, classifies threats by risk level, and generates forensic incident reports — all automated.

[![View Project](https://img.shields.io/badge/🔗_View_on_GitHub-Linux_HIDS_Automation-DC2626?style=flat-square&logo=github&logoColor=white)](https://github.com/karthikbiju-sys/linux-hids-automation)

**How it works:**

```python
# Linux HIDS — Core Detection Flow
[Log Ingestion]  →  /var/log/auth.log  +  syslog  +  kern.log
      ↓
[Pattern Matching]  →  SSH failures  |  Repeated IPs  |  Anomalies
      ↓
[Risk Engine]  →  LOW  |  MEDIUM  |  HIGH  classification
      ↓
[Incident Report]  →  report.txt  (forensic output, timestamped)
      ↓
[Active Monitor]  →  Persistent real-time watch mode
```

| Feature | Description |
|:---|:---|
| 🔐 **SSH Brute-Force Detection** | Tracks repeated failed logins per IP across `auth.log` |
| 👁️ **Real-Time Threat Monitoring** | Persistent watch mode — listens for host anomalies continuously |
| 🎯 **Risk Classification Engine** | Severity scoring: LOW / MEDIUM / HIGH per detected event |
| 📋 **Forensic Incident Reporting** | Auto-generates structured `report.txt` with timestamps & IP data |
| 📂 **Multi-Log Analysis** | Parses `auth.log`, `syslog`, and kernel logs simultaneously |
| 🚀 **Phase-Based Execution** | Demo ingestion → Active monitoring, cleanly separated |
